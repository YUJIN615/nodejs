예외 처리
- 예외(exception): 처리하지 못한 에러고
- 노드 스레드를 멈추게 함
- 노드는 기본적으로 싱글 스레드라 스레드가 멈춘다는 것은 프로세스가 멈추는 것
- 에러 처리는 필수
- 노드는 에러 처리가 중요하다.

- 기본적으로 try catch 문으로 예외를 처리
- 에러가 발생할 만한 곳을 try catch로 감쌈

- 노드 비동기 메서드의 에러는 따로 처리하지 않아도 됨
- 콜백 함수에서 에러 객체 제공
- 에러가 나도 멈추지는 않지만 에러 객체를 로그에 남겨서 확인할 수 있음

- 프로미스의 에러는 catch 안 붙여도 멈추지 않음. 경고 메세지가 뜸
- 노드 버전이 올라가면 catch 안 붙였을때 에러날 수 있으니 꼭 붙여주기

- 프로세스 객체에 'unCaughtException' event 를 달아서 모든 에러를 한 곳에서 처리할 수 있음
- 하지만 콜백 함수의 동작이 보장되지 않음
- 따라서 복구 작업용으로 쓰는 것은 부적합
- 에러 내용 기록용으로만 쓰는게 좋음
